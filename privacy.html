Privacy Policy

Come gestiamo i tuoi dati quando usi EmailChainGuard.

How we handle your data when you use EmailChainGuard.

Ultimo aggiornamento: 19 maggio 2026
Last updated: May 19, 2026
In sintesi: EmailChainGuard analizza le email che apri direttamente nel tuo browser/dispositivo. Al nostro server inviamo solo i nomi di dominio (mai email complete), per verificarne reputazione e configurazione tecnica. Tutti gli altri dati (mittenti conosciuti, conversazioni, preferenze) restano nel tuo profilo Microsoft 365.

1. Chi siamo

EmailChainGuard ("noi", "il servizio") è un add-in per Microsoft Outlook progettato per rilevare tentativi di spoofing del dominio e impersonificazione nelle email. Questa policy descrive come trattiamo i dati personali ai sensi del Regolamento UE 2016/679 (GDPR).

Per domande o richieste: info@notmyfriend.onmicrosoft.com

2. Quali dati raccogliamo

2.1 Dati che restano sul tuo dispositivo

I seguenti dati sono memorizzati esclusivamente nel tuo profilo Microsoft 365 (Roaming Settings) o nel browser (localStorage). Non vengono mai inviati ai nostri server:

  • Indirizzi email dei mittenti con cui hai già scambiato messaggi
  • Partecipanti delle conversazioni email (per il rilevamento di nuovi CC)
  • Domini della tua organizzazione configurati nelle impostazioni
  • Preferenze utente (lingua, attivazione funzioni)
  • Token di accesso a Microsoft Graph (se hai attivato la lettura conversazione completa)
  • Preferenza di consenso agli analytics anonimi (opt-in, default disattivata)

2.2 Dati inviati al nostro server di analisi

Quando apri un'email, il plugin invia al nostro backend solo:

  • I nomi di dominio presenti nell'email (es. esempio.com) — mai indirizzi email completi
  • Un identificativo opaco della conversazione (non collegabile alla tua identità)

Questi dati sono utilizzati esclusivamente per eseguire i controlli WHOIS, DNS e reputazione del dominio. Non vengono salvati permanentemente. I log temporanei del server (fino a 7 giorni) contengono solo metadati tecnici (timestamp, codice di stato HTTP).

2.2.2 Dati inviati al servizio di monitoraggio errori

Quando il plugin incontra un errore tecnico (crash, eccezione non gestita, fallimento di chiamate al backend o a Microsoft Graph), invia automaticamente a Sentry uno stack trace e alcuni metadati per consentirci di diagnosticarlo e correggerlo. Vedi la sezione 10 per il dettaglio completo. Questa funzione è attiva di default ma puoi disattivarla dalle impostazioni del plugin.

2.3 Dati richiesti a Microsoft Graph (opzionale)

Se hai attivato la funzione "Lettura conversazione completa", il plugin chiede a Microsoft Graph i metadati delle email precedenti dello stesso thread (mittenti, CC, oggetto, data). Questi dati vengono elaborati solo nel tuo browser e mai inviati al nostro server.

3. Base giuridica del trattamento

Il trattamento dei dati di dominio è basato sull'interesse legittimo (art. 6.1.f GDPR): proteggere te e la tua organizzazione da tentativi di frode e attacchi BEC. L'attivazione di Microsoft Graph è invece basata sul tuo consenso esplicito (art. 6.1.a GDPR), revocabile in qualsiasi momento dalle impostazioni del plugin.

Anche il monitoraggio errori (vedi sezione 10) si basa sull'interesse legittimo (art. 6.1.f GDPR): mantenere il plugin sicuro e funzionante. Puoi esercitare in qualsiasi momento il diritto di opposizione (art. 21 GDPR) disattivando il toggle nelle impostazioni, oppure scrivendoci all'email di supporto.

4. Conservazione dei dati

  • Dati locali: conservati nel tuo profilo finché non li elimini con "Cancella memoria mittenti" o disinstalli l'add-in
  • Log del server: max 7 giorni, poi cancellazione automatica
  • Token Graph: validi al massimo 1 ora, rinnovati silenziosamente quando autorizzato

5. Condivisione con terze parti

Per analizzare i domini, il backend interroga i seguenti servizi esterni, ai quali vengono comunicati esclusivamente i nomi di dominio (mai email o dati personali):

  • WHOIS registries – per la data di registrazione del dominio
  • Server DNS pubblici – per verificare record MX, SPF, DMARC
  • VirusTotal (privacy policy) – per la reputazione del dominio
  • Google Safe Browsing (privacy policy) – per il check su URL malevoli
  • Umami Cloud (privacy policy) – statistiche d'uso aggregate, solo se hai dato consenso esplicito dalle impostazioni del plugin. Server hosted in UE. Nessun cookie, nessun identificatore persistente, nessun dato personale.
  • Sentry GmbH (privacy policy) – monitoraggio errori tecnici del plugin. Server hosted in UE (Francoforte). Stack trace e metadati tecnici anonimizzati lato client prima dell'invio. Attivo di default ai sensi dell'interesse legittimo (art. 6.1.f), opt-out disponibile nelle impostazioni del plugin.

Non condividiamo dati personali con inserzionisti, broker di dati o altre terze parti per scopi commerciali.

6. Trasferimenti extra-UE

Il nostro server è ospitato presso fornitori cloud con data center in Europa (Render.com – Frankfurt). VirusTotal e Google Safe Browsing sono servizi statunitensi: i loro trasferimenti sono coperti dalle Standard Contractual Clauses della Commissione Europea.

7. I tuoi diritti

In qualsiasi momento puoi esercitare i seguenti diritti:

  • Accesso: sapere quali dati abbiamo (per il server: praticamente nessuno, vista la non conservazione)
  • Cancellazione: cancellare la memoria locale dalle impostazioni del plugin
  • Revoca consenso: disattivare Microsoft Graph dalle impostazioni
  • Portabilità: i tuoi dati sono già nel tuo profilo Microsoft, esportabili tramite Outlook
  • Reclamo: all'Autorità Garante per la Protezione dei Dati Personali (garanteprivacy.it)

8. Sicurezza

Tutte le comunicazioni tra il plugin e il backend avvengono in HTTPS con TLS 1.2+. L'accesso al backend richiede un'API key. Il token Microsoft Graph è memorizzato solo nel tuo profilo Microsoft 365 e non lascia mai il tuo dispositivo.

9. Cookie e analytics

EmailChainGuard non utilizza cookie. La memorizzazione delle preferenze avviene tramite Microsoft Roaming Settings e localStorage del browser, che non sono cookie e non sono utilizzati per profilazione.

Analytics anonimi (opt-in) — Dalle impostazioni del plugin puoi attivare l'invio di statistiche d'uso aggregate al nostro fornitore di analytics Umami Cloud (server in UE). Questa opzione è disattivata di default e richiede il tuo consenso esplicito (art. 6.1.a GDPR), revocabile in qualsiasi momento dalle stesse impostazioni.

Quando il consenso è attivo, raccogliamo solo i seguenti eventi anonimi e aggregati, senza alcun identificatore persistente:

  • Avvio del plugin (versione, lingua dell'interfaccia)
  • Tipo di banner mostrato (primo contatto, nuovo partecipante, dominio sospetto) e, per i domini sospetti, una fascia di rischio (bassa / media / alta)
  • Cambio lingua (it ↔ en)
  • Apertura impostazioni
  • Attivazione/disattivazione della lettura conversazione completa (Microsoft Graph)
  • Aggiunta/rimozione di un dominio della tua organizzazione (solo conteggio totale, mai il nome del dominio)
  • Cancellazione memoria mittenti
  • Click su "Segnala un problema" o "Suggerisci una funzione"
  • Modifica del consenso analytics stesso

Non raccogliamo mai: indirizzi email, mittenti, oggetti delle email, domini analizzati, indirizzi IP completi, cookie, fingerprint del browser, identificatori utente persistenti. Umami non utilizza cookie e non rileva l'utente individualmente tra sessioni.

10. Monitoraggio errori

Per mantenere il plugin sicuro e funzionante, gli errori tecnici (crash JavaScript, eccezioni non gestite, fallimenti delle chiamate al backend o a Microsoft Graph) vengono inviati automaticamente al servizio di error monitoring Sentry (server in UE — Francoforte). Questa funzione è attiva di default sulla base dell'interesse legittimo (art. 6.1.f GDPR) e puoi disattivarla in qualsiasi momento dalle impostazioni del plugin (toggle "Monitoraggio errori") esercitando il diritto di opposizione (art. 21 GDPR).

Quando il monitoraggio è attivo, raccogliamo solo i seguenti dati tecnici:

  • Stack trace dell'errore (file, riga, colonna, nome funzione)
  • Tipo e messaggio dell'eccezione (dopo lo scrubbing automatico delle stringhe sensibili)
  • Versione del plugin, lingua dell'interfaccia, fase logica in cui l'errore si è verificato (es. "oauth_dialog_message_parse", "backend_analyze_timeout")
  • User-agent del browser e timestamp dell'evento
  • Breadcrumb tecnici (sequenza di eventi che hanno preceduto l'errore, es. click su un pulsante, fetch verso il backend), anch'essi scrubati

Prima dell'invio, il plugin esegue uno scrubbing automatico aggressivo lato client che rimuove:

  • Indirizzi email (sostituiti con [redacted-email])
  • Token JWT e Bearer di Microsoft Graph (sostituiti con [redacted-token])
  • ID opachi di messaggi e conversazioni Graph (sostituiti con [redacted-graph-id])
  • Headers Authorization, Cookie e la nostra API key (rimossi interamente)
  • Body delle richieste HTTP (rimossi: potrebbero contenere la lista dei domini analizzati)
  • Qualsiasi evento o breadcrumb relativo al flusso OAuth (URL login.microsoftonline.com, auth-callback.html) è droppato interamente

Inoltre, Sentry è configurato con:

  • sendDefaultPii: false — non vengono inviati IP utente, cookie, o headers identificativi
  • Session Replay disabilitato — niente registrazione del DOM né dei click
  • Performance tracing disabilitato — niente trace delle pagine visitate o degli URL richiesti
  • Profiling disabilitato — niente profilazione delle prestazioni del browser
  • Console capture disabilitato — i messaggi console.log non vengono mai inviati

I dati raccolti sono conservati da Sentry per un massimo di 30 giorni (retention del piano Developer), poi cancellati automaticamente. Vengono utilizzati esclusivamente per diagnosticare e correggere bug. Non vengono mai usati per profilazione, marketing, o terze parti.

11. Modifiche a questa policy

Eventuali aggiornamenti sostanziali saranno notificati tramite il plugin stesso e tramite la pagina "Cosa c'è di nuovo" nelle impostazioni. La data di ultimo aggiornamento è sempre visibile in cima a questo documento.

12. Limitazioni del servizio

EmailChainGuard è uno strumento di supporto alla sicurezza. Non sostituisce il giudizio umano e non garantisce il rilevamento di tutti i tentativi di attacco. In caso di dubbio sull'autenticità di un'email, verifica sempre con il presunto mittente attraverso un canale alternativo (telefono, video chiamata).

1. Who we are

EmailChainGuard ("we", "the service") is a Microsoft Outlook add-in designed to detect domain spoofing and impersonation attempts in emails. This policy describes how we process personal data under EU Regulation 2016/679 (GDPR).

For questions or requests: info@notmyfriend.onmicrosoft.com

2. What data we collect

2.1 Data that stays on your device

The following data is stored exclusively in your Microsoft 365 profile (Roaming Settings) or in the browser (localStorage). It is never sent to our servers:

  • Email addresses of senders you have already exchanged messages with
  • Participants in email conversations (for new CC detection)
  • Your organization's domains configured in the settings
  • User preferences (language, feature activation)
  • Microsoft Graph access tokens (if you have enabled full conversation reading)
  • Anonymous analytics consent preference (opt-in, default disabled)

2.2 Data sent to our analysis server

When you open an email, the plugin sends to our backend only:

  • The domain names present in the email (e.g. example.com) — never full email addresses
  • An opaque conversation identifier (not linkable to your identity)

This data is used exclusively to perform WHOIS, DNS and domain reputation checks. It is not permanently stored. Temporary server logs (up to 7 days) contain only technical metadata (timestamp, HTTP status code).

2.2.2 Data sent to the error monitoring service

When the plugin encounters a technical error (crash, unhandled exception, failure of calls to the backend or to Microsoft Graph), it automatically sends to Sentry a stack trace and some metadata so we can diagnose and fix it. See section 10 for full details. This feature is enabled by default but you can disable it from the plugin settings.

2.3 Data requested from Microsoft Graph (optional)

If you have enabled the "Full conversation reading" feature, the plugin asks Microsoft Graph for metadata of previous emails in the same thread (senders, CCs, subject, date). This data is processed only in your browser and never sent to our server.

3. Legal basis for processing

Processing of domain data is based on legitimate interest (art. 6.1.f GDPR): to protect you and your organization from fraud attempts and BEC attacks. Microsoft Graph activation is instead based on your explicit consent (art. 6.1.a GDPR), revocable at any time from the plugin settings.

Error monitoring (see section 10) is also based on legitimate interest (art. 6.1.f GDPR): keeping the plugin secure and functional. You can exercise your right to object (art. 21 GDPR) at any time by disabling the toggle in the settings, or by emailing our support address.

4. Data retention

  • Local data: kept in your profile until you delete it with "Clear sender memory" or uninstall the add-in
  • Server logs: max 7 days, then automatic deletion
  • Graph tokens: valid for max 1 hour, silently renewed when authorized

5. Sharing with third parties

To analyze domains, the backend queries the following external services, to which only domain names are communicated (never emails or personal data):

  • WHOIS registries – for the domain registration date
  • Public DNS servers – to verify MX, SPF, DMARC records
  • VirusTotal (privacy policy) – for domain reputation
  • Google Safe Browsing (privacy policy) – for malicious URL checks
  • Umami Cloud (privacy policy) – aggregated usage statistics, only if you have given explicit consent in the plugin settings. Servers hosted in the EU. No cookies, no persistent identifiers, no personal data.
  • Sentry GmbH (privacy policy) – technical error monitoring of the plugin. Servers hosted in the EU (Frankfurt). Stack traces and technical metadata anonymized client-side before sending. Enabled by default under legitimate interest (art. 6.1.f), opt-out available in the plugin settings.

We do not share personal data with advertisers, data brokers or other third parties for commercial purposes.

6. Non-EU transfers

Our server is hosted by cloud providers with data centers in Europe (Render.com – Frankfurt). VirusTotal and Google Safe Browsing are US services: their transfers are covered by the European Commission's Standard Contractual Clauses.

7. Your rights

At any time you can exercise the following rights:

  • Access: know what data we hold (for the server: practically none, given non-retention)
  • Deletion: clear local memory from the plugin settings
  • Consent withdrawal: disable Microsoft Graph from the settings
  • Portability: your data is already in your Microsoft profile, exportable via Outlook
  • Complaint: to your local Data Protection Authority

8. Security

All communications between the plugin and the backend use HTTPS with TLS 1.2+. Backend access requires an API key. The Microsoft Graph token is stored only in your Microsoft 365 profile and never leaves your device.

9. Cookies and analytics

EmailChainGuard does not use cookies. Preferences are stored via Microsoft Roaming Settings and browser localStorage, which are not cookies and are not used for profiling.

Anonymous analytics (opt-in) — From the plugin settings you can enable sending aggregated usage statistics to our analytics provider Umami Cloud (EU-hosted servers). This option is disabled by default and requires your explicit consent (art. 6.1.a GDPR), revocable at any time from the same settings.

When consent is active, we only collect the following anonymous and aggregated events, with no persistent identifier:

  • Add-in load (version, interface language)
  • Type of banner shown (first contact, new participant, suspicious domain) and, for suspicious domains, a risk bucket (low / medium / high)
  • Language change (it ↔ en)
  • Settings opened
  • Enabling/disabling full conversation reading (Microsoft Graph)
  • Adding/removing an organization domain (only total count, never the domain name)
  • Sender memory cleared
  • Click on "Report a problem" or "Suggest a feature"
  • Change of the analytics consent itself

We never collect: email addresses, senders, email subjects, analyzed domains, full IP addresses, cookies, browser fingerprints, persistent user identifiers. Umami does not use cookies and does not track individual users across sessions.

10. Error monitoring

To keep the plugin secure and functional, technical errors (JavaScript crashes, unhandled exceptions, failures of calls to the backend or to Microsoft Graph) are automatically sent to the error monitoring service Sentry (EU-hosted servers — Frankfurt). This feature is enabled by default on the basis of legitimate interest (art. 6.1.f GDPR) and you can disable it at any time from the plugin settings ("Error monitoring" toggle), exercising your right to object (art. 21 GDPR).

When monitoring is active, we only collect the following technical data:

  • Stack trace of the error (file, line, column, function name)
  • Type and message of the exception (after automatic scrubbing of sensitive strings)
  • Plugin version, interface language, logical phase in which the error occurred (e.g. "oauth_dialog_message_parse", "backend_analyze_timeout")
  • Browser user-agent and event timestamp
  • Technical breadcrumbs (sequence of events that preceded the error, e.g. button clicks, fetch to the backend), also scrubbed

Before sending, the plugin performs aggressive automatic client-side scrubbing that removes:

  • Email addresses (replaced with [redacted-email])
  • JWT and Bearer tokens from Microsoft Graph (replaced with [redacted-token])
  • Opaque Graph message and conversation IDs (replaced with [redacted-graph-id])
  • Authorization, Cookie headers and our API key (removed entirely)
  • HTTP request bodies (removed: they may contain the list of analyzed domains)
  • Any event or breadcrumb related to the OAuth flow (URLs login.microsoftonline.com, auth-callback.html) is dropped entirely

Furthermore, Sentry is configured with:

  • sendDefaultPii: false — no user IP, cookies, or identifying headers are sent
  • Session Replay disabled — no DOM or click recording
  • Performance tracing disabled — no traces of visited pages or requested URLs
  • Profiling disabled — no browser performance profiling
  • Console capture disabled — console.log messages are never sent

The collected data is retained by Sentry for a maximum of 30 days (Developer plan retention), then automatically deleted. It is used exclusively to diagnose and fix bugs. It is never used for profiling, marketing, or third parties.

11. Changes to this policy

Any substantial updates will be notified through the plugin itself and through the "What's new" page in the settings. The date of last update is always visible at the top of this document.

12. Service limitations

EmailChainGuard is a security support tool. It does not replace human judgment and does not guarantee detection of all attack attempts. If in doubt about the authenticity of an email, always verify with the presumed sender through an alternative channel (phone, video call).
